## 2. AI 동료가 회사 데이터에 닿는 원리 (MCP) (20분)

> 방금 써 본 **NotebookLM · Deep Research · Idea Generation**은 **Google이 만든** 에이전트였습니다. 이런 에이전트가 **우리 회사 데이터·시스템**에 닿는 원리가 바로 **MCP**입니다. 이제 그 원리를 보고, 직접 **No-Code로 내 도구를 묶은 에이전트**를 만듭니다.

**한 줄 개념**

- **두뇌 = LLM** : 생각·판단
- **팔다리 = Tool** : 행동·정보 수집
- **신경 = MCP** : 두뇌가 팔다리를 호출하는 표준 통신 규약 (USB-C 같은 표준)

### 에이전트 vs MCP — 헷갈리지 않기

한 줄로: **에이전트는 '일하는 주체(수행비서)', MCP는 그 비서가 도구에 연결되는 '표준 규격'**입니다. 둘은 경쟁이 아니라 **포함 관계** — 에이전트가 MCP를 *씁니다*.

- **에이전트(Agent)** = 두뇌(LLM) + 지시(역할·목표) + 골라 묶은 도구 + 기억 + 실행 루프(PLAN→DELEGATE→SYNTHESIZE→RESPOND). 무엇을 할지 스스로 정하고 도구를 골라 실행하는 **'몸 전체'**.
- **MCP** = 그 에이전트가 **도구에 닿는 표준 연결**. 비유의 **'신경'**. USB-C처럼 어떤 도구든 같은 방식으로 꽂힙니다.

| 구분 | 에이전트(Agent) | MCP |
|---|---|---|
| 정체 | 일하는 주체 — 수행비서 | 도구 연결 규약 — 표준 케이블 |
| 범위 | LLM + 지시 + 도구 + 기억 + 루프 (**전체**) | 에이전트와 도구 사이의 **한 계층** |
| 비유 | 비서라는 '사람' 전체 | 비서가 전화·시스템에 꽂는 표준 단자 |
| 개수 | 한 에이전트가 **여러** MCP 도구 사용 | 한 MCP 도구를 **여러** 에이전트가 공유 |
| 핵심 질문 | "**누가** 일하는가" | "도구에 **어떻게** 연결되는가" |

> 헷갈림 방지 — **MCP를 붙였다고 에이전트가 되는 게 아닙니다.** 에이전트는 LLM+지시+도구+루프 *전체*이고, MCP는 그중 **도구 연결** 부분의 표준입니다.

### 오늘의 도구 카탈로그 — 어디에 끼울까

> **머릿속 그림**: 에이전트(LLM) 가운데에 두고, 주변 도구들이 USB 슬롯처럼 꽂혀 있습니다. 각 에이전트는 **필요한 것만** 골라 끼워요.

```
                  ┌────────────────────────────┐
   Gmail ────┐    │                            │   ┌── DART · 회사 기본정보
   Drive ────┼────│       에이전트 (LLM)        │───┼── DART · 최근 공시 조회
   Calendar ─┘    │   PLAN→DELEGATE→SYNTH→RESP │   ├── DART · 위험 신호 탐지
                  │                            │   ├── DART · 재무 추세 분석
                  └──────────────┬─────────────┘   └── DART · 임원 현황
                                 │
                       사내 지식 데이터스토어
                       (사규·매뉴얼·FAQ + 출처)
```

| 종류 | 도구 | 쓰임 | 와우 포인트 |
|---|---|---|---|
| 📚 **데이터스토어** | 사내 지식 (Vertex AI Search) | 사규·매뉴얼·FAQ 검색 | **답변에 인용 출처가 자동으로** 붙음 — 감사·법무가 좋아함 |
| 🔌 **Workspace 커넥터** | Gmail | 메일 조회·초안 생성 | 발송 전 검토 단계 → 위험 0 |
| 🔌 **Workspace 커넥터** | Drive | 파일 읽기·문서 저장 | 산출물이 **내 Drive에 그대로** |
| 🔌 **Workspace 커넥터** | Calendar | 일정 조회 | "내일 미팅" 자동 인식 |
| 🧩 **DART MCP** | 회사 기본정보 | 회사명 → CEO·종목코드·산업·설립일·홈페이지 | 한국 상장사 **실시간 공식 정보** |
| 🧩 **DART MCP** | 최근 공시 조회 | 최근 N일 공시 목록 (제목·날짜·URL) | **어제 공시가 그대로** — 인덱싱 불가 영역 |
| 🧩 **DART MCP** | 위험 신호 탐지 | 공시에서 카테고리별 위험 키워드 매칭 + 위험도 점수 | **원본에 없는 파생 지표** (1~10점) |
| 🧩 **DART MCP** | 재무 추세 분석 | 연간 매출·영업이익 + **YoY 증감률 자동 계산** | 검색이 아닌 **계산**의 결과 |
| 🧩 **DART MCP** | 임원 현황 | 사업보고서 기준 임원 명단·임기·경력 | 분기마다 자동 업데이트 |

> **적재적소 한 줄** — 문서 검색·출처 인용은 **데이터스토어**, 실시간 회사 데이터·계산·액션은 **MCP**. 에이전트는 필요한 것만 골라 묶습니다.

> **오늘 만나볼 마법** — *위험도 점수 8.7/10* 같은 숫자는 **DART 원본 어디에도 없습니다**. MCP 서버가 공시 100건을 보고 가중치 합산해 만들어내는 *파생 지표*예요. **이게 문서 검색(데이터스토어)과 함수 호출(MCP)의 본질적 차이**입니다.

### 커넥터(데이터스토어) vs Custom MCP — 언제 무엇을?

> **한 줄 직관** — 데이터스토어는 **"책에서 찾아 줘"**(문서에 적힌 것), Custom MCP는 **"시스템 두드려 가져와 줘"**(운영 DB 안의 살아있는 값). 같은 일이 아니라 **다른 계층**입니다. 섹션 3-0에서 같은 질문을 두 도구에 던져 직접 대결시켜 볼 거예요.
>
> 자주 나오는 질문: *"BigQuery·Storage에 데이터를 넣는 커넥터가 있는데, 굳이 Custom MCP를 만들 필요가 있나요?"* → 위의 한 줄 직관이 답이에요. 운영 DB의 *현재 값*은 인덱싱으로 풀리지 않습니다.

| | 커넥터 → 데이터스토어 (BigQuery·GCS) | Custom MCP |
|---|---|---|
| 하는 일 | 데이터를 **미리 인덱싱**해 **검색·그라운딩**(읽기) | 내 함수를 **실시간 호출** — 정밀 조회·로직·액션 |
| 잘하는 것 | 문서·자료 의미 검색·요약, **출처 자동 인용** | 키로 정확히 조회, 외부 API, 쓰기/발송, 비즈니스 로직 |
| 데이터 | 내가 미리 넣어둔 내 회사 데이터 | 내 시스템이든 **외부**든, 실시간 |
| 결과 | "관련 있는 내용"(검색 결과) | "정해진 형식의 정확한 값"(함수 반환) |
| 코드 | No-Code(연결만) | 함수 작성·배포 필요 |

**Custom MCP가 꼭 필요한 경우** (아니면 커넥터로 충분):

- **외부 데이터·API** — 외부 회사·뉴스처럼 내 DB에 없는 것. 커넥터로 불가 → MCP가 답.
- **실시간·정밀 조회** — "거래처 X의 최근거래일" 같은 키 기반 정확한 값(검색이 아니라 조회).
- **액션** — 메일 발송·티켓 생성·레코드 수정 등 '읽기'가 아닌 '행동'.
- **외부·레거시 시스템** — 30+ 커넥터에 없는 ERP·사내 마이크로서비스.
- **재사용·거버넌스** — 한 도구를 여러 에이전트가 표준으로 공유, 원본 DB 대신 함수 결과만 노출.

> **결론** — 내 문서·데이터를 **검색·요약**만 하면 → **커넥터(데이터스토어)**가 더 간단(출처 자동). **외부 API·실시간·정밀 조회·액션**이면 → **Custom MCP**. 오늘 *외부 회사 검색*이 바로 커넥터로는 못 해서 MCP가 필요한 대표 사례입니다.

**판별 팁**

- 답변에 **실제 회사 이름·날짜·금액·점수**가 구체적으로 나오고 도구 호출 흔적이 보이면 → **MCP 정상**.
- 특히 위험도 점수(예: `6.2/10`)나 YoY 증감률(예: `+12.4%`) 같은 **파생값**이 나오면 → MCP가 *계산까지* 잘 도는 상태.
- "정보를 찾을 수 없다" → 비상장사이거나 회사명이 정확하지 않음. *정식 회사명* 사용 (예: "삼성전자" — "삼전" X).
- 첫 호출이 5~10초 느린 건 정상 (DART corpCode 캐시 로딩). 이후 호출은 빠릅니다.

### 🧭 에이전트 만들기 — 공통 5단계 (한 번만 익히면 끝)

> 🎯 **결정적 한 방** — 위 위험도 점수 `2.5/10`은 **DART 원본 공시 어디에도 적혀 있지 않습니다**. MCP 서버가 *공시 제목*을 *카테고리별 위험 키워드*에 매칭하고 *가중치 합산*해서 만들어낸 **파생 지표**예요. 데이터스토어로는 흉내조차 못 냅니다.

**🧠 한 줄로 기억하기**

| | 데이터스토어 (커넥터로 인덱싱) | Custom MCP |

> **느낀 점 한 줄** — *"문서 검색은 일반 챗봇도 흉내냅니다. 운영 시스템에서 실시간 값을 가져오는 건 Custom MCP가 있어야만 가능해요."* — 이게 엔터프라이즈 AI의 진짜 분기점입니다. 이제 #1·#2로 가서 둘을 **묶어** 진짜 워크플로우를 만듭니다.

### 3-1. 에이전트 #1 — 🚀 빅테크 미팅 사전 브리핑 자동 생성

> **WOW** — *"내일·모레 NVIDIA, 삼성전자, 네이버, MS, SK하이닉스, 셀트리온 미팅이 있는데 사전 브리프 좀 만들어줘"* — 한 문장에. **Calendar가 6개 미팅을 자동 인식 → 각 회사를 DART에서 실시간 분석 → Drive에 회사별 1쪽 브리프 6개 저장 → 본부장 보고 Gmail 초안까지**. 단일 챗봇은 절대 못 합니다 — 이게 **Gemini Enterprise**의 진짜 가치예요.

**왜 이게 *Gemini Enterprise라야* 가능한가** — 이 에이전트는 **5가지 도구를 동시에** 묶습니다:
- 🗓️ **Calendar** — 내일 미팅 6건 자동 추출 (제목·참석자·장소)
- 📧 **Gmail** — 본부장 사전 지시 메일 / 거래처 사전 자료 요청 메일 자동 참고
- 📊 **DART MCP × 5도구** — 각 한국 상장사(삼성·네이버·SK하이닉스·셀트리온)별 어제 공시·재무·위험 신호
- 📚 **사내 지식 데이터스토어** — 『미팅 사전 브리핑 체크리스트 §1』 § 자동 인용
- 📁 **Drive** — `미팅 브리프 2026/[날짜]` 폴더 자동 생성 → 회사별 문서 6개 저장
- 📨 **Gmail Draft** — 본부장 보고용 메일 초안 (Drive 링크 포함)

DART API만 있으면 일반 챗봇이나 단순 스크립트로도 흉내가 됩니다. 하지만 **"내 캘린더의 내일 미팅을 보고, 본부장 메일을 읽고, 우리 회사 매뉴얼대로 정리해서, 내 Drive에 저장하고, 본부장 보고 메일까지"** — 이건 *Workspace + 데이터스토어 + Custom MCP 통합*만이 만들 수 있는 합성품이에요.

**사전 준비 확인** — 본인 Google Calendar에 D+1 (월) ~ D+2 (화)에 6개 빅테크 미팅이 등록돼 있어야 합니다 (`samples/calendar_meetings.md` 참조 — 운영진이 사전 등록). Gmail 받은편지함에는 본부장 사전 지시 메일도 있어요 (`samples/gmail_inbox.md`).

**Before / After 한눈에**

| | 지금까지 (Before) | 이 에이전트 후 (After) |
|---|---|---|
| 누가 하나 | 본인이 새벽까지 / 영업기획팀에 요청 | 본부장이 출근길에 채팅창 한 번 |
| 거치는 단계 | Calendar 보고 → 6개 회사 각각 검색 → DART 사이트 6번 방문 → 엑셀 정리 → PPT 6개 → Drive 업로드 → 본부장 메일 작성 | 채팅창 한 번 |
| 소요 시간 | **2~3시간 × 6개 = 반나절 이상** | **약 1~2분 (전체 6개 일괄)** |
| 깊이 | 회사별 깊이 편차 큼 | **6개 회사 동일 양식 + 매뉴얼 §1 6섹션 강제** |

- **가로지르는 워크플로우**: 미팅 식별(Calendar) → 메일 컨텍스트(Gmail) → 회사별 실시간 분석(DART × 6) → 매뉴얼 적용(데이터스토어) → 산출물 저장(Drive) → 보고(Gmail Draft)

## 4. 정리 & 월요일 액션 플랜 (10분)

### 오늘의 핵심 3줄

1. Gemini Enterprise는 회사 데이터·도구에 연결돼 **실제 일을 하는 'AI 동료'** — 검색 답변이 아니라 *실행 결과*.
2. **문서는 데이터스토어(출처 자동), 도구는 MCP** — 적재적소로 표준화하면 누가 써도 같은 품질.
3. 진짜 가치는 "도입했다"가 아니라 **업무 워크플로우에 녹아드는 것** — 한 명이 만든 에이전트를 **팀 전체가 같은 품질로 사용**.

### 🗓️ 월요일 아침 3단계 — 회사로 가져갈 짐 챙기기

> 오늘 본 그림을 **실제 우리 회사**에서 어떻게 출발시킬지 3단계로 압축했습니다.

| 단계 | 누가 | 무엇을 | 결과물 |
|---|---|---|---|
| **① 도착 첫날** | 본인 | 내 업무 흐름 한 줄 + 가장 답답한 1지점 + 거기 붙일 도구 1개 적기 | **포스트잇 한 장** |
| **② 첫 주 안** | IT/플랫폼팀과 30분 | 그 도구가 **커넥터로 충분**한지 **Custom MCP 필요**한지 결정 | **기술 선택지 1줄** |
| **③ 2주 안** | 본인 + IT | 그 도구를 **하나** 사전 준비 → **No-Code로 1개 에이전트** 만들어 5명에게 공유 | **돌아가는 첫 사내 에이전트** |

> **No-Code의 의미** — 데이터·도구·인증의 *기초공사*는 IT가, 그 위에 얹는 *워크플로우*는 현업이. 이 분업이 가능해진 게 오늘 만난 변화의 핵심입니다.

### 내 워크플로우에서 MCP 후보 찾기 (3분 워크시트)

> 옆 사람과 1분씩 나눠 채워 보세요.

1. **내 업무 흐름 한 줄 (AS-IS)** — 예: "고객 문의 접수 → 분류 → 담당 배정 → 응대"
2. **가장 답답한 지점** (6지점 중 하나) — 정보수집 / 분류·요약 / 판단·우선순위 / 연결·전달 / 고객대응 / 학습·개선
3. **거기 붙일 도구 1개** — 커넥터(검색·요약)면 데이터스토어, 실시간·정밀 조회·액션이면 MCP

> 이 한 줄이 여러분 조직의 **첫 AX 워크플로우 과제**입니다. (좋은 과제는 "AI가 할 수 있는 일"이 아니라 "워크플로우를 바꾸면 성과가 달라지는 지점"에서 나옵니다.)

### 와우 포인트 회고 — 오늘 내가 느낀 순간은?

> 가장 인상 깊었던 순간에 √ 표시해 보세요. 회사 보고서·기획서에서 이 표현이 가장 잘 통합니다.

- [ ] 한국 상장사 에이전트가 **DART의 어제·오늘 공시**를 진짜로 가져와 보여줬을 때
- [ ] 답변에 **위험도 점수(예: 3.5/10)** 가 떴는데, 그게 *DART 원본엔 없는 MCP 계산값*임을 깨달았을 때
- [ ] 미팅 브리핑 에이전트가 **내 캘린더의 NVIDIA·삼성·네이버 미팅 6건을 자동 인식**했을 때
- [ ] 외국계(NVIDIA·MS)에 대해 **DART 폴백 메시지**가 명시적으로 떴을 때 (한계를 정직하게 알려줌)
- [ ] **본부장 지시 메일이 자동으로 반영**돼 *"미팅 1시간 전까지"* 같은 요구사항이 브리프에 들어갔을 때
- [ ] Drive에 **회사별 1쪽 PDF 6장 + 폴더**가 자동 생성됐을 때
- [ ] 본부장 회신용 **Gmail Draft가 이미 본문 채워서** 열렸을 때
- [ ] 사내 매뉴얼 §1·§2·§3이 **자동으로 인용**되며 회사별 양식이 같았을 때
- [ ] 내가 만든 에이전트를 **공유 링크로 옆자리에 30초 만에** 전달했을 때

**토론 질문**
- 미팅 사전 브리핑 에이전트 — 우리 조직에서 *이 작업을 매일 새벽까지 하던 사람*은 누구였을까? (사업개발 / 영업본부 / 비서실 / 임원)
- *Calendar × Gmail × Drive × DART × 데이터스토어 5중 통합* 중 우리 조직에서 **가장 빠진 1개**는 뭐일까? (사내 매뉴얼이 데이터스토어 인덱싱이 안 됨 / Custom MCP가 없음 / Workspace 안 씀)
- 같은 패턴으로 또 무엇을 만들 수 있을까? (예: *주간 임원 보고 자동 생성* — Calendar + Gmail + Drive + DART / *컨퍼런스 사후 정리* — Calendar 회의록 + Gmail 명함 + DART 회사 정보 / *RFP 자동 매칭* — 외부 입찰공고 + 사내 솔루션 카탈로그)
- 사내 정책 안내 에이전트 — 어디서 가장 답답했나? (인사 / 총무 / IT 헬프데스크)
- "데이터스토어(우리 회사의 판단 기준) × MCP(실시간 외부 데이터)" 한 쌍 — 우리 회사에 또 어떤 *판단 기준 문서*가 있나? (구매 가이드 / 채용 기준 / 보안 정책 / 신용평가 매뉴얼 / 투자 위원회 가이드)
