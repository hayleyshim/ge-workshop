## 오늘 만나볼 'AI 동료', Gemini Enterprise

Gemini Enterprise는 똑똑한 AI 비서를 넘어, **우리 회사의 일하는 방식과 데이터에 맞춰 움직이는 AI 동료**입니다. 일반 AI가 인터넷의 공개 정보로 답한다면, Gemini Enterprise는 회사의 문서·메일·일정까지 함께 찾아 **여러분 업무에 딱 맞는 답**을 줍니다. 마치 수년간 같이 일한 동료처럼요.

### **일반 Gemini와 Gemini Enterprise, 무엇이 다른가요?**

가장 큰 차이점은 **'커넥터'** 와 **'보안'** 입니다. 일반 Gemini가 인터넷상의 공개된 정보를 바탕으로 답변하는 **'지식 전문가'** 라면, Gemini Enterprise는 회사의 내부 시스템과 직접 연결되어 실제 업무를 수행하는 **'액션 전문가'** 에 가깝습니다.

| 구분 | 일반 Gemini | Gemini Enterprise |
|---|---|---|
| 지식의 범위 | 웹, 책 등 공개된 데이터 | 공개 데이터 + 회사 내부 데이터 (문서, 이메일, 캘린더 등) |
| 주요 기능 | 정보 검색, 번역, 글쓰기, 아이디어 구상 | 내부 정보 검색, 문서 요약/작성, 이메일 초안, 회의 일정 조율 |
| 커넥터 | 제한적 | Google Drive, Calendar, Email 등 회사 시스템과 직접 연동 |
| 보안 | 일반 사용자용 보안 | 강화된 기업용 보안 및 데이터 관리 정책 적용 |
| 핵심 가치 | 범용적인 지식 제공 | 업무 생산성 극대화 및 시간 절약 |


### **오늘 직접 느껴보실 3가지 순간**

1. **"진짜 조직의 데이터로 답이 나오네"** — 인터넷 검색이 아닌 연동된 데이터스토어에 근거한 답 + 출처(소스)까지.
2. **"코드 한 줄 없이 나만의 AI 비서를 만들었어"** — 대화만으로 5분 만에 생성한 에이전트.
3. **"내가 만든 걸 옆자리 동료가 바로 쓸수있네"** — 개인이 생성한 에이전트를 조직에 그대로 공유.


### **성공적인 AI 도입을 위한 필수 조건: 사용자의 역할과 노력**

물론, Gemini Enterprise를 도입하는 것만으로 모든 것이 해결되고 AI 경험이 저절로 가속화되는 것은 아닙니다. 최고의 스포츠카가 있어도 운전법을 모르면 무용지물이듯, Gemini Enterprise의 진정한 가치는 사용자인 우리가 AI를 이해하고 적극적으로 활용하려는 노력과 만났을 때 비로소 발현됩니다.

성공적인 AI 기반 업무 혁신을 위해 다음과 같은 노력이 함께 필요합니다.

- AI와 친해지기: AI를 '어려운 기술'이 아닌 '똑똑한 동료'로 생각하고, 간단한 업무부터 꾸준히 사용하며 그 가능성을 체험하고 이해해야 합니다.

- 데이터 정돈하기: AI는 결국 데이터를 기반으로 작동합니다. 내가 가진 파일, 폴더, 기록 등을 AI가 더 잘 이해하고 학습할 수 있도록 명확한 제목과 체계적인 구조로 정리하는 노력이 필요합니다.

- 'AI 활용 노하우' 습득하기: 나의 업무를 더 빠르고 효율적으로 처리하기 위해 AI에게 어떻게 질문하고, 어떤 데이터를 보여주고, 어떤 결과물을 요구할지 등 자신만의 AI 활용 노하우를 적극적으로 고민하고 학습해야 합니다.

Gemini Enterprise는 여러분이 AI 시대를 향해 나아갈 수 있도록 돕는 강력한 엔진입니다. 이 엔진에 '데이터'라는 연료를 공급하고, '활용 노하우'라는 운전 기술을 익혀, 업무의 새로운 고속도로를 달려나갈 준비를 함께 시작해 봅시다.


> 오늘은 연결·인증·세팅 같은 어려운 준비가 **이미 다 끝나 있습니다.** 여러분은 **레고 블록 조립**만 하면 돼요. 90분 동안 그 첫 경험을 압축해서 같이 해봅니다.

## 1. 아이스브레이킹 — 인사부터 나누기

새 동료와 친해지려면 가벼운 대화부터 시작하는게 좋겠죠? Gemini에게 여러분을 소개해 주세요.

### 1-1. Google 검색 켜기

채팅창 하단의 조절 버튼을 눌러 **Google 검색**을 **켭니다(ON)**.

<img src="images/02-google-search-toggle.png" alt="Google 검색 토글 켜기" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

*▲ 도구 버튼을 누르고 'Google 검색'을 켜면 실시간 웹 정보를 근거로 답합니다.*

### 1-2. 자기소개

`OOO` 부분만 본인에 맞게 바꿔서 붙여넣기:

```
안녕하세요! 제 이름은 OOO입니다.
저는 OOO 업무를 하고, 최신 AI·AX 트렌드에 관심이 많아요.
기억해주세요.
```
<img src="images/02-memory.png" alt="Gemini Enterprise 로그인 화면" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

<img src="images/02-memory2.png" alt="Gemini Enterprise 로그인 화면" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

> 💡 "기억해주세요"라고 하면 Gemini가 이 내용을 **저장된 메모리**에 담아 둡니다. 새 채팅을 열어도 "저는 OOO 예요"라고 매번 다시 말할 필요가 없어요. 좌측 메뉴 **"저장된 메모리 관리"** 에서 무엇이 저장됐는지 직접 확인하고 지울 수도 있습니다. 한글로 "기억해주세요" 라고 하면 요청한 정보가 한글로 입력됩니다.


### 1-3. 맞춤 정보 요청

나를 파악한 Gemini에게 물어보세요:

```
제가 관심있어 할 만한 최신 기술·산업 뉴스가 있다면
소스와 함께 알려주세요.
```

답변이 실시간으로 생성되는 과정을 볼 수 있고, 답변 하단의 **소스**를 누르면 어떤 자료를 근거로 답했는지 확인됩니다.

<img src="images/03-answer-generation.png" alt="실시간 답변 생성 화면" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

*▲ Gemini가 추론 과정을 보여주며 실시간으로 답변을 생성합니다. 하단의 '소스'로 출처를 확인하세요.*

> **잠깐, 왜 '소스(출처)'가 중요할까요?** 일반 AI는 배운 내용만으로 답하다 보니 가끔 틀린 정보를 그럴듯하게 말합니다(이걸 '환각(hallucination)'이라고 해요). Gemini Enterprise는 **회사 데이터나 실시간 검색을 근거로** 답하고, 답변 아래 **"소스" 버튼**으로 무엇을 보고 답했는지 보여줍니다. 그래서 훨씬 믿을 수 있어요. AI 답변은 항상 출처를 확인하는 습관을 들이면 좋습니다.

### 1-4. 내 업무 문제 등록 ★

> **왜 지금 할까요?** 이후 실습에서 직접 만들 에이전트를 *남이 만든 시나리오*가 아닌 **내 문제**로 완성하기 위해서입니다. 지금 생각해보면 에이전트 생성 실습에서 적용해볼 수 있어요.

아래 빈칸을 채워서 Gemini에게 보내세요: 예) 일정 확인, 자료 작성 등

```
제 업무에서 매주 반복되는데 시간이 가장 많이 걸리는 일은 [**직접 작성해보세요**] 입니다.
이걸 AI 동료가 도와준다면 어떤 식으로 가능할지 아이디어를 알려주세요.
```

> 💡아직 답이 완벽하지 않아도 됩니다. **3. 에이전트 생성하기** 실습에서 실제 에이전트로 다시 만들어볼 거예요. **오늘 워크숍이 끝나면 여러분 손에 '내 문제를 푸는 에이전트 워크플로우 초안' 하나가 남습니다.**

## 2. 커넥터로 회사 데이터에 연결하기

**이제 진짜 핵심입니다 — Gemini Enterprise가 우리 회사 고유의 데이터(사규·거래처 정보 등)에 연동되는 방법**

여기서 조직의 **데이터**를 끌어오는 통로가 **커넥터(= 데이터스토어)**이고, 그 위에서 실제 작업을 수행하는 것이 **도구**입니다([공식 문서](https://docs.cloud.google.com/gemini/enterprise/docs/connectors/custom-mcp-server/set-up-custom-mcp-server))

> 🧰 **잠깐 — 세 가지 개념만 숙지해주세요** *(처음엔 가볍게 읽고 넘어가도 됩니다)*
> - **에이전트** = 일하는 비서. 무엇을 할지 정하고 필요한 도구를 골라 씁니다.
> - **커넥터(= 데이터 스토어)** = 자료 창고로 통하는 문. 사규·메일·공시 같은 데이터를 Gemini Enterprise 안으로 끌어옵니다. 콘솔에서 **'데이터 스토어'** 메뉴로 등록하는게 전부 이겁니다.
> - **도구** = 비서의 손발. 커넥터(데이터 스토어)에서 검색하거나, 메일을 쓰거나, 공시를 가져오는 실제 작업이에요.
>
> 👉 한 줄로: **에이전트는 커넥터로 조직의 데이터에 연결하고 도구로 작업한다.**


에이전트에게 어떤 **도구**를 쥐여주느냐에 따라 할 수 있는 일이 달라져요. 그 도구들은 **커넥터**라는 통로를 통해 우리 데이터·시스템에 연결됩니다. 

아래는 **에이전트가 골라 쓰는 도구** 관점의 목록입니다. (콘솔에서는 모두 '커넥터(데이터 스토어)'로 등록돼요 — 자세한 건 아래 박스를 참고하세요.)


| 커넥터 | 종류 | 무엇을 하나 | 언제 좋은가 |
|---|---|---|---|
| 📚 **사내 지식** | 검색용 **데이터 스토어**(문서 색인) | 사규·매뉴얼·FAQ를 검색 | **"규칙·절차가 어떻게 돼 있지?"** — 답에 출처가 자동으로 붙어요 |
| 🔌 **Gmail / Drive / Calendar** | Google **기본 커넥터** | 메일 읽고 초안 쓰기 / 파일 읽고 저장 / 일정 보기 | **"내 메일·일정·파일을 봐줘"** |
| 🧩 **기업 공시 도구** | **Custom MCP 서버** | 글로벌 상장사(미국 SEC)의 실시간 공시·재무 가져오기 | **"이 회사 지금 상황이 어때?"** — 어제 올라온 공시까지 |

>
> 👉 셋 중 **'Custom MCP 서버'는 기업 공시 하나뿐**입니다. 나머지 둘은 문서 색인용 데이터 스토어와 구글 기본 커넥터예요. 종류는 다르지만 **등록은 같은 메뉴('데이터 스토어')로** 하고, Gemini Enterprise UI에서 모두 **커넥터로 똑같이** 붙어서 사용법은 동일합니다.

> 💬 **MCP(Model Context Protocol)가 뭐예요?** *'AI에게 외부 시스템(공시·ERP·CRM 등)을 연동하는 **표준 방식**'* 정도로만 알면 충분합니다. 오늘은 미리 만들어 환경에서 **선택하기만** 하면 돼요.

<img src="images/06-connector-list.png" alt="채팅창 하단 원통 아이콘의 도구 목록" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

*▲ 채팅창 하단 '원통' 아이콘을 누르면 쓸 수 있는 커넥터 목록이 보입니다. 필요한 것만 켜서 씁니다.*

> **딱 하나만 기억하세요** — 문서에 **적혀 있는 것**(사규·매뉴얼)은 📚 **사내 지식(사규)**으로, 시스템에서 **실시간으로 가져와야 하는 값**(오늘 올라온 공시 등)은 🧩 **기업 공시 같은 전용 도구**로. 커넥터 목록에서 필요한 항목만 선택하세요.

### 2-2. 기업 공시 도구(Custom MCP 서버)에 직접 물어보기

사내 지식이 '우리 문서'를 찾아준다면, 이번엔 외부 시스템에서 **실시간으로 값을 가져오는** 커넥터를 써봅니다.
새 거래처·공급사·투자처를 검토할 때, "이 회사 요즘 괜찮은가?"를 공식 공시로 즉시 확인해볼 수 있습니다.
채팅창에서 커넥터만 켜면 바로 체험됩니다.

**2-2-1.** 새 채팅창 하단 **'원통' 아이콘**에서 **🧩 기업 공시 도구**만 **켭니다(ON)**. 

**2-2-2.** 글로벌 상장사 이름을 넣어 물어보세요:

엔비디아 최근 90일 공시를 날짜, 유형과 함께 알려줘.

> 💡 엔비디아 대신 애플·마이크로소프트·테슬라 등 검토하려는 회사 이름을 넣으세요.
> 비상장·미국 외 미등록 기업은 조회되지 않습니다. 첫 호출은 5~10초 느릴 수 있어요.

✅ **결과 — 이렇게 판단하세요**
- **최근 공시**에 실적 발표(8-K)·정기보고서(10-K/10-Q)가 정상적으로 올라오고 있는가 → 정상 운영 중
- **위험 신호** 점수가 낮고 특이 이벤트가 없는가 → 리스크 낮음
- **매출·순이익**이 성장 또는 안정 추세인가 → 재무 건전

이 세 가지가 확인되면, 그 회사를 거래처로 검토할 **기초 근거**가 마련된 겁니다.
어떤 사내 문서에도 안 적힌, 공식 공시에서 실시간으로 가져와야만 알 수 있는 값이라는 게 사내 지식과 다른 점이에요.

- ⚠️ "찾을 수 없음" → 회사명 표기 확인(비상장·외국계는 조회 불가).
- ⚠️ 이 결과는 1차 스크리닝용입니다. 실제 거래 결정 전에는 10-K/10-Q 원문의 재무제표와 외부 신용평가를 함께 확인하세요.

> 🎯 **두 커넥터(데이터 스토어)를 다 써 봤어요** 
> — 📚 **사내 지식**(우리 문서에서 찾기)과 🧩 **기업 공시**(외부 시스템에서 실시간으로 가져오기). 
> 3번에서는 이 커넥터(데이터 스토어)들을 **에이전트에 연결**해, 매번 켜지않아도 알아서 골라 쓰는 **나만의 비서**를 만듭니다.

### 3-2. 두 번째 에이전트 — 주간 업계 동향 브리핑 (스케줄 작업)

> ⏰ **이번엔 '스케줄 작업'을 걸어봅니다.** 앞의 정책 도우미가 "물어보면 답하는" 비서였다면, 이 에이전트는 **매주 알아서 우리 업계·경쟁사 뉴스를 조사해 요약 리포트를 나에게 보내주는** 비서예요.

> ⚠️ **왜 '나에게' 보낼까요?** 스케줄 작업은 사람이 중간에 확인하지 못하는 **자동 실행**입니다. 그래서 리포트는 **에이전트 사용자 본인 앞으로만** 보냅니다. 이렇게 하면 다른 사람이 관여하는 작업(승인 대기)과 달리 자동으로 실행됩니다.

> 제목·굵기 같은 서식이 그대로 살아 변환됩니다. *(Gemini Enterprise의 Drive 커넥터는 표준 액션상 '파일 업로드'만 지원해, 에이전트가 직접 .docx나 Google Docs 문서를 만들지는 않습니다. 문서 생성이 꼭 필요하면 회사에 돌아가 IT팀과 커스텀 액션으로 확장할 수 있어요.)*

<img src="images/22-new-agent-drive2.png" alt="사내 지식에서 찾아 출처와 함께 답하는 예시" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

> **결과** — *"사용자 Gmail 계정의 캘린더 일정에 등록된 회사들과의 미팅이 자동으로 인식돼서 각 회사를 기업 공시 도구로 분석한 정리본이 Drive에 저장됐어."* 커넥터를 묶으면 이렇게 **하나의 흐름(workflow)**가 만들어집니다.

### 3-4. 내 문제로 바꿔보기(심화)

> 1-4에서 적어둔 **내 업무의 반복 문제**를 꺼내, 에이전트로 설계해 봅니다.

아래 프롬프트의 `[ ]` 부분을 1-4에서 적었던 내용으로 채워서 붙여넣으세요:

```
아까 제가 적은 '매주 반복되는 일: [직접 작성해보세요]'를 도와줄 에이전트를 만들고 싶어요.
어떤 커넥터(사내 지식 / Gmail / Drive / Calendar / 기업 공시 중)를 연결하면 될지,
그리고 에이전트 설정 칸에 뭐라고 적으면 될지 같이 설계해줘.
```

Gemini Enterprise가 커넥터 조합과 설정 초안을 제안하면, 그대로 **+ 새 에이전트**에 연결하여 실제로 만들어 보세요.

> ✅ **결과** — 에이전트 이름에 내 이름·업무가 들어가 있고, 테스트 칸에 내 실제 질문을 던졌을 때 의미 있는 답이 나오면 성공입니다.
> 💡 **시간이 부족하다면** 설계 초안만 저장해도 됩니다.

---

### 3-5. 👥 내가 만든 에이전트, 팀에 공유하기

> **중요 포인트** — 한 명이 만든 에이전트를 **팀 전체가 같은 품질로 사용할 수 있어요**. 개인 챗봇과 결정적으로 다른 부분이에요.

1. 좌측 에이전트 목록에서 방금 만든 에이전트 클릭 → 우측 상단 **공유(Share)** 아이콘
2. **사용자 또는 그룹** 입력 → 권한 선택 (보기 / 사용 / 편집)
3. **공유 링크 복사** → 옆자리 동료에게 전송 → 동료가 클릭 → **그대로 동작** ✅

<img src="images/23-new-agent-share.png" alt="에이전트 공유 버튼 위치" style="max-width:100%;border:1px solid #E2E8F0;border-radius:8px;margin:.6em 0">

*▲ 우측 상단 '공유' 버튼으로 링크를 만들면, 동료가 같은 에이전트를 바로 쓸 수 있습니다.*

> **느낀 점 한 줄** — *"IT 지식이 없어도 우리 팀 전용 AI 동료(에이전트)를 배포했다."* 게다가 누가 무엇을 만들었는지, 어떤 커넥터를 쓰는지 모두 **관리자 콘솔에서 관리**됩니다.

## 4. 정리 

### 오늘의 핵심 3줄

1. Gemini Enterprise는 회사 데이터·도구에 연결돼 **실제 일을 하는 'AI 동료'** — 검색 답변이 아니라 *실행 결과*를 줍니다.
2. **문서에 적힌 건 사내 지식(출처 자동), 실시간 값은 전용 도구(Custom MCP 서버, 예: SEC 공시·DART)** — 알맞게 골라 쓰면 누가 써도 같은 품질. 필요한 시스템은 Custom MCP 서버로 계속 연동할 수 있어요.
3. 진짜 가치는 **업무에 활용하여 생산성을 높이는 것** — 한 명이 만든 에이전트를 **팀 전체가 같은 품질로** 사용 가능합니다.
